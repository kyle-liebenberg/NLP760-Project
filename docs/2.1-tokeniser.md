# Subword Tokenizer (BPE) 

## Related files:
- `src/train_tokenizer.py`
- `models/isizulu_bpe_tokenizer.json` 

## Overview:
A custom Byte-Pair Encoding (BPE) tokenizer trained specifically on the 261 articles in the dataset.
The vocabulary size is strictly limited to 5,000 tokens. Unseen words encountered during testing or deployment will be assigned an `[UNK]` (unknown) token to prevent pipeline crashes.

## Purpose:
It breaks down raw isiZulu text into smaller, mathematically digestible numerical IDs. Instead of splitting by whole words, it splits by "subwords" (chunks of characters).
isiZulu is a highly agglutinative language where prefixes and suffixes are attached to root words to convey meaning (e.g., subject concords, object concords). Standard word-level tokenization would treat every morphological variation as a completely new word, leading to a massive, sparse, and inefficient vocabulary. BPE captures these roots and affixes perfectly.
The algorithm starts with a base vocabulary of individual characters. It scans the entire dataset and iteratively merges the most frequently adjacent pairs of characters into single tokens. It stops when it reaches the target vocabulary size of 5,000.

## Libraries:
- `tokenizers` (by HuggingFace): This library is built in Rust, making it fast, and it natively supports creating robust BPE pipelines from scratch.
