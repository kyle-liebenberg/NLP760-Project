# 2.2 Dataset splitting (`src/preprocessing.py`)

This script loads the raw authorship CSV and produces **stratified train / validation / test splits** that preserve each author's relative share of the data. It does not tokenize text or build NumPy arrays — that is handled by [`src/tokenizer.py`](2.1-tokeniser.md).

## Purpose

Authorship attribution needs fixed, reproducible partitions so that:

- Every author appears in every split (where possible).
- No article appears in more than one split (no data leakage).
- Class proportions stay similar across train, validation, and test.

## Inputs

| Path | Description |
|------|-------------|
| `data/raw/isizulu_authors_dataset.csv` | Raw scrape output; must contain `author`, `url`, and `text` columns |

## Split strategy

A **two-stage stratified split** on the `author` column:

| Stage | Operation | Result |
|-------|-----------|--------|
| 1 | 70% train / 30% holdout | `train_df`, `holdout_df` |
| 2 | Holdout split 50/50 | `val_df` (15%), `test_df` (15%) |

**Hyperparameters** (top of script):

| Parameter | Value | Meaning |
|-----------|-------|---------|
| `RANDOM_STATE` | `42` | Reproducible shuffling |
| `TEST_SIZE` | `0.15` | 15% of all rows → test |
| `VAL_SIZE` | `0.15` | 15% of all rows → validation |

### Current split sizes (last run)

| Split | Articles | % of total |

| Train | 184 | 70.0% |
| Validation | 39 | 14.8% |

All 11 authors are present in train, validation, and test. Author counts per split are recorded in `outputs/metrics/split_metadata.json`.

## Main functions

| Function | Role |
|----------|------|
| `load_data` | Reads CSV and validates required columns |
| `stratified_split` | Performs the two-stage split with assertions on row counts and author coverage |
| `verify_split` | Prints a verification table; checks author presence, proportions, and URL overlap between splits |
| `save_splits` | Writes `train.csv`, `val.csv`, `test.csv` |
| `save_split_metadata` | Writes `split_metadata.json` with sizes and per-split author counts |

### Leakage checks

`verify_split` uses `url` as a unique article key:

- **Zero overlap** between train, val, and test URL sets is required (assertion fails otherwise).
- Warnings are printed if any author is missing from val or test (possible with very small per-author counts, e.g. Simangaliso Ntshangase).

## Outputs

| File | Description |
|------|-------------|
| `data/splits/train.csv` | 70% stratified training set |
| `data/splits/val.csv` | 15% validation set |
| `data/splits/test.csv` | 15% test set |
| `outputs/metrics/split_metadata.json` | Split sizes, `random_state`, per-split author counts |

Each CSV retains the original columns: `author`, `url`, `text`.

## Usage

From the project root:

```bash
python3 src/preprocessing.py
```

Or programmatically:

```python
from src.preprocessing import run
train_df, val_df, test_df = run()
```
