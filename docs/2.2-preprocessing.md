## Overview:
The data transformation script that bridges the gap between raw text and model-ready numerical matrices.
The sequence length for the neural network is strictly padded or truncated to 500 tokens. The train/test split is set to 80/20 and uses stratification.

## Purpose:
It converts author names into integers, splits the dataset into training and testing sets, converts text into numerical sequences, and normalizes the length of every sequence.
Neural networks (like the CNN-LSTM) cannot process raw text; they require numerical tensors of a strictly fixed shape. Furthermore, traditional machine learning models and deep learning models must be trained and evaluated on the exact same data splits to ensure scientific fairness. Stratification ensures that all 11 authors are equally represented in both the training and testing sets, preventing class imbalance biases.
It utilizes a Label Encoder to map string names to integers (0 to 10). It splits the data, preserving the class distribution. It then loops through every article, applies the BPE tokenizer to get integer sequences, and uses a padding utility to append `[PAD]` tokens to the end of short articles, or chops off tokens from articles exceeding 500 subwords.

## Libraries used:
- `scikit-learn` for the LabelEncoder and `train_test_split utilities`. 
- `tensorflow.keras.preprocessing.sequence` for the highly efficient `pad_sequences` function
- `numpy` to save the final tensors for rapid loading during model training.
