# Traditional baselines

## Related files:
- `src/baselines.py`

## Overview:
Benchmark models using traditional machine learning algorithms: Logistic Regression and a Support Vector Machine (LinearSVC).
The SVM achieved a remarkable weighted F1-Score of 84.19%. The models use character n-grams (ranging from 2 to 5 characters) rather than standard word frequencies.

## Purpose:
It establishes a performance floor. It provides a mathematical benchmark that the deep learning architecture must attempt to surpass.
In NLP, it is a standard practice to start simple. If a computationally cheap SVM performs exceptionally well, a computationally expensive CNN-LSTM is only justified if it yields better results or better explainability. Character n-grams were chosen because they act as a "poor man's subword tokenization," effectively capturing the prefixes and suffixes of the agglutinative isiZulu language.
It converts the raw training text into a TF-IDF (Term Frequency-Inverse Document Frequency) matrix. This matrix weighs how important a specific character chunk (like uku or nga) is to a specific author relative to the whole dataset. It then trains the Logistic Regression and SVM classifiers to draw mathematical boundaries between the 11 authors based on these frequencies.

## Libraries used:
- `scikit-learn` for the `TfidfVectorizer`, the classification algorithms (`LogisticRegression`, `LinearSVC`), and the evaluation metrics. It is the industry standard for traditional machine learning workflows.
