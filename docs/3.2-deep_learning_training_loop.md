
## 3.2. Deep Learning Training Loop (`src/train_cnn_lstm.py`)

### Overview

The optimization execution script responsible for reading data artifacts, loading model graphs, governing forward/backward optimization runs, and measuring generalization errors.

### Important Technical Details

* **Optimization Criteria:** Evaluated using categorical macro-averaged F1-score to track predictive balance evenly across all classes, mitigating slight imbalances in author block representation.
* **Early Stopping Anchor:** Restores model weights to the historical optimum achieved at **Epoch 39** out of 40.

### What it Does

It sets up the batch processing infrastructure, drives weight optimization via backpropagation over time, saves the top-performing network weights, and prints comprehensive validation arrays.

### Why it Does it

Neural networks require disciplined optimization loops to prevent divergence and over-indexing. On a low-resource dataset, letting a network run unconstrained for 40 epochs would result in it memorizing the training subset. This wrapper acts as a supervisor, dynamically adjusting learning parameters and cutting training as soon as validation errors begin to plateau.

### How it Does it

1. **Hyperparameter Ingestion:** Reads `tokenizer_metadata.json` to dynamically extract vocabulary size and input constraints.
2. **Data Tensor Loading:** Ingests the NumPy matrices (`X_train.npy`, `X_val.npy`, etc.) directly into memory for fast CPU/GPU batch caching.
3. **Training Governance Callbacks:**
* `EarlyStopping(patience=8)`: Monitors validation loss. If the validation loss fails to improve for 8 consecutive steps, it halts training to halt overfitting.
* `ReduceLROnPlateau(patience=3)`: If the validation loss flatlines for 3 epochs, it scales down the learning rate by 50% (`factor=0.5`), shrinking step sizes to perform microscopic weight optimizations.


4. **Evaluation Engine:** Converts final raw probability distributions into distinct class predictions via an `argmax` reduction, comparing them to the test array to generate exact precision-recall profiles.

### Libraries Used

* `numpy`: For lightning-fast tensor array data streams.
* `sklearn.metrics`: To generate unbiased multi-class performance profiles (`classification_report`, `f1_score`).
