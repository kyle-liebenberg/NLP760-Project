# 3.2 Deep learning training loop (`src/train_cnn_lstm.py`)

This script loads the padded BPE arrays and label tensors, builds the CNN-LSTM from [`cnn_lstm_model.py`](3.1-deep_learning_model_architecture.md), trains with validation-based early stopping, and reports test-set metrics.

## Purpose

Neural models need a controlled optimisation loop: fixed data tensors, checkpointing of the best validation weights, learning-rate adaptation, and final evaluation on the held-out test split. This script ties together tokenizer outputs and the hybrid architecture into one reproducible training run.

## Prerequisites

Run these steps first (or use [`run_pipeline.py`](../run_pipeline.py) through phase 2):

1. [`src/preprocessing.py`](2.1-preprocessing.md): CSV splits
2. [`src/tokenizer.py`](2.2-tokeniser.md): `X_*.npy`, `y_*.npy`, `tokenizer_metadata.json`

| Required artifact | Path |
|-------------------|------|
| Training tensors | `data/splits/X_train.npy`, `y_train.npy` |
| Validation tensors | `data/splits/X_val.npy`, `y_val.npy` |
| Test tensors | `data/splits/X_test.npy`, `y_test.npy` |
| Hyperparameters | `outputs/metrics/tokenizer_metadata.json` |

## Training configuration

| Setting | Value | Source |
|---------|-------|--------|
| Epochs (max) | 40 | Script constant |
| Batch size | 32 | Script constant |
| Optimiser | Adam, lr=0.001 | `build_cnn_lstm` |
| Random seed | 19 (Keras) + TF op determinism | Top of script |

Dimensions are read dynamically from metadata:

- `vocab_size` ← `actual_vocab`
- `max_seq_length` ← `max_sequence_length`
- `num_classes` ← `num_classes`
- `label_classes` ← `classes` (for classification report)

## Callbacks

| Callback | Settings | Role |
|----------|----------|------|
| `EarlyStopping` | `monitor='val_loss'`, `patience=8`, `restore_best_weights=True` | Stop when validation loss plateaus; revert to best epoch |
| `ModelCheckpoint` | `monitor='val_loss'`, `save_best_only=True` | Persist best weights to disk |
| `ReduceLROnPlateau` | `factor=0.5`, `patience=3`, `min_lr=1e-5` | Halve learning rate if validation loss stalls |

## Main flow

1. Load `tokenizer_metadata.json` and NumPy arrays.
2. Call `build_cnn_lstm(vocab_size, max_seq_length, num_classes)`.
3. `model.fit` on train with `validation_data=(X_val, y_val)` and callbacks.
4. Predict on `X_test`; compute macro F1, precision, recall, accuracy.
5. Save metrics to `outputs/metrics/dl_report.json`.

## Outputs

| File | Description |
|------|-------------|
| `models/cnn_lstm_best.keras` | Best checkpoint by validation loss |
| `outputs/metrics/dl_report.json` | Test-set macro metrics |

## Evaluation metric

**Primary metric:** macro-averaged F1 on the test set, consistent with baselines so results are directly comparable across model families.

## Dependencies

- `tensorflow`, `keras`
- `numpy`
- `scikit-learn` (`classification_report`, `f1_score`, etc.)

## Design choices

1. **Metadata-driven shapes**: Retraining the tokeniser updates sequence length and vocab automatically without editing the training script.
2. **Validation-driven stopping**: Test data is never used for hyperparameter decisions; only for the final report.
3. **Checkpoint + restored weights**: `EarlyStopping` with `restore_best_weights=True` ensures evaluation uses the lowest-validation-loss epoch, not necessarily the last epoch.
