# 2.1 Preprocessing and splitting (`src/preprocessing.py`)

This script loads the raw authorship CSV, **augments** long articles by overlapping word chunks, and produces **stratified train / validation / test splits** that preserve each author's relative share of the data. It does not tokenize text or build NumPy arrays, that is handled by [`src/tokenizer.py`](2.2-tokeniser.md).

## Purpose

Authorship attribution on full newspaper articles yields few training examples per author. Chunking expands the effective sample count for the neural model while keeping author labels. Stratified splitting then ensures:

- Every author appears in every split (where possible).
- Class proportions stay similar across train, validation, and test.
- Reproducible partitions for baselines, BPE training, and the CNN-LSTM.

## Prerequisites

| Path | Description |
|------|-------------|
| `data/raw/isizulu_authors_dataset.csv` | Raw scrape output; must contain `author`, `url`, and `text` columns |

Run [`src/build_author_dataset.py`](1-data.md) first if this file is missing.

## Data augmentation (chunking)

Long articles are sliced into overlapping **word** windows before splitting:

| Parameter | Value | Meaning |
|-----------|-------|---------|
| `CHUNK_SIZE` | 150 | Maximum words per chunk |
| `CHUNK_OVERLAP` | 30 | Words shared between consecutive chunks |

**Rules** (`chunk_text_data`):

- Articles with ≤ 150 words are kept as a single row (unchanged `url`).
- Longer articles are walked with stride `chunk_size - overlap` (120 words).
- Chunks shorter than 40 words are discarded.
- Chunk URLs append `#chunk0`, `#chunk1`, … to the parent article URL for traceability.

On the current corpus (~877 articles), this expands the dataset to **2,040 chunks** before splitting.

## Split strategy

A **two-stage stratified split** on the `author` column:

| Stage | Operation | Result |
|-------|-----------|--------|
| 1 | 70% train / 30% holdout | `train_df`, `holdout_df` |
| 2 | Holdout split 50/50 | `val_df` (15%), `test_df` (15%) |

| Parameter | Value | Meaning |
|-----------|-------|---------|
| `RANDOM_STATE` | `42` | Reproducible shuffling |
| `TEST_SIZE` | `0.15` | 15% of all chunks → test |
| `VAL_SIZE` | `0.15` | 15% of all chunks → validation |

## Main functions

| Function | Role |
|----------|------|
| `load_data` | Reads CSV and validates required columns |
| `chunk_text_data` | Slices long articles into overlapping word chunks |
| `stratified_split` | Two-stage stratified split on `author` |
| `verify_split` | Prints chunk counts and per-author distribution table |
| `save_splits` | Writes `train.csv`, `val.csv`, `test.csv` |
| `save_split_metadata` | Writes `split_metadata.json` with chunking and split sizes |
| `run` | Orchestrates the full preprocessing pipeline |

| File | Description |
|------|-------------|

Each CSV retains the columns: `author`, `url`, `text`.

## Pipeline position

```
build_author_dataset.py  →  preprocessing.py  →  tokenizer.py  →  baselines.py  →  train_cnn_lstm.py
         (raw CSV)           (CSV chunks)         (NumPy + BPE)     (TF-IDF)          (CNN-LSTM)
```

## Design choices

1. **Chunk before split**: Augmentation happens on the full corpus so train/val/test all contain chunks from the same underlying articles, stratified by author rather than by chunk ID.
2. **Word-level windows**: Simple whitespace splitting matches downstream BPE (whitespace pre-tokeniser) and keeps implementation transparent.
3. **Overlapping chunks**: 30-word overlap reduces boundary effects where an author’s style might span a single cut point.
