# 3.1 Deep learning model architecture (`src/cnn_lstm_model.py`)

This module defines the **hybrid CNN-LSTM** classifier used for isiZulu authorship attribution. It consumes padded BPE token-ID sequences from [`tokenizer.py`](2.2-tokeniser.md) and outputs an 11-way softmax over journalists.

## Purpose

Classical baselines treat documents as unordered bags of n-grams. The neural model instead learns **local subword patterns** (via convolution) and **order-sensitive structure** (via LSTM), while a global pooling skip path retains document-level salience similar to TF-IDF’s strongest features.

On agglutinative isiZulu text, author style often appears in recurring morphemes and phrasing rhythms rather than isolated dictionary words alone.

## Architecture overview

The model uses the Keras **Functional API** (not `Sequential`) so two branches can be merged after shared convolutional features:

```
Input (batch, max_seq_len)
    → Embedding(vocab_size, 128)
    → SpatialDropout1D(0.3)
    → Conv1D(128, k=5, relu) + BatchNorm
           ├─► MaxPool1D(2) → LSTM(64) → Dropout ──┐
           └─► GlobalMaxPooling1D → Dropout ───────┤
                                            concatenate
                                                → Dense(64, relu) + BatchNorm + Dropout(0.3)
                                                 → Dense(11, softmax)
```

## Hyperparameters (defaults)

| Argument | Default | Role |
|----------|---------|------|
| `embedding_dim` | 128 | Dense subword vector size |
| `filters` | 128 | Conv1D output channels |
| `kernel_size` | 5 | Subword n-gram window for morphology |
| `lstm_units` | 64 | Recurrent hidden size |
| `dropout_rate` | 0.4 | Dropout after LSTM and global branch |

`vocab_size`, `max_seq_length`, and `num_classes` are passed in at build time from `tokenizer_metadata.json` (currently vocab **3000**, length **404**, classes **11**).

## Layer-by-layer behaviour

| Layer | Role |
|-------|------|
| **Embedding** | Maps integer BPE IDs to 128-d vectors |
| **SpatialDropout1D (0.3)** | Drops entire embedding dimensions across time steps to reduce phrase memorisation |
| **Conv1D + BatchNorm** | Sliding 5-subword filter; extracts local morphological/stylistic features |
| **Track A - LSTM** | `MaxPooling1D(2)` downsamples time; `LSTM(64)` models sequential dependencies |
| **Track B - Global skip** | `GlobalMaxPooling1D` captures strongest activations anywhere in the document (lexical “highlights”) |
| **Concatenate** | Merges 64-d LSTM state with 128-d global features → 192-d representation |
| **Dense head** | 64-unit ReLU block with batch norm and dropout; 11-way softmax output |

## Compilation

| Setting | Value |
|---------|-------|
| Optimizer | `Adam(learning_rate=0.001)` |
| Loss | `sparse_categorical_crossentropy` (integer author labels) |
| Metric | `accuracy` |

## Main function

| Function | Role |
|----------|------|
| `build_cnn_lstm(...)` | Constructs and compiles the Functional model; returns a Keras `Model` |

## Usage

The model is not run directly. [`src/train_cnn_lstm.py`](3.2-deep_learning_training_loop.md) imports `build_cnn_lstm` and supplies dimensions from saved metadata.

```python
from src.cnn_lstm_model import build_cnn_lstm

model = build_cnn_lstm(
    vocab_size=3000,
    max_seq_length=404,
    num_classes=11,
)
```

## Design choices

1. **Dual track**: Combines sequence modelling with a global max-pool bypass so the network is not forced to compress all stylistic signal through the LSTM bottleneck alone.
2. **Spatial dropout**: Regularises along the embedding dimension, which is often more effective than independent element dropout on sequence inputs.
3. **Modest capacity**: ~520k parameters (see `model.summary()` at train time) suits a low-resource, chunked corpus without matching large-language-model scale.
